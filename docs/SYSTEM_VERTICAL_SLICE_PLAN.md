# 全体縦切り計画

決定日: 2026-07-20
方針: 個別の周期最適化やA〜Fの詳細試験を先に完璧にするのではなく、まず2台・全センサ・記録・監視・安全停止が接続された全体形を早く作る。その後に実機計測で細部を最適化する。

## 目標となる最初の全体形

```text
制御側 XIAO ESP32S3                         通信・記録側 XIAO ESP32S3 Sense
────────────────────                        ──────────────────────────────
BNO08X / VL53L5CX / INA226                  GNSS / 比較用BNO08X
PCA9685 / サーボ / VESC                      SoftAP Web UI / 状態表示
安全状態機械                                 microSD BIN/TXT 記録
        │                                              │
        └──── UART 921600 bps 双方向 ───────────────────┘
              telemetry → SD保存       STOP/E-STOP → 制御側
```

この段階で完成を求めるのは「全経路が一度つながること」である。状態推定、時刻同期の精度評価、ウェイポイント、ロール／舵／推進の閉ループ制御、周期の最適化は後段に置く。

## 通信・記録側の確定配線候補

既存の実装を照合した結果、次の割当はピン競合がない。

| 機能 | XIAO Sense端子 | 既存の動作実績 |
| --- | --- | --- |
| GNSS GT-505GGBL5-DR-N UART 115200 | RX=D0、TX=D1 | BNO+GNSS+SDプロジェクト |
| 比較用BNO08X専用I2C 100 kHz | RST=D2、INT=D3、SDA=D4、SCL=D5 | BNO+GNSS+SDプロジェクト |
| 制御側UART 921600 | RX=D7、TX=D6 | UART→SD保存成功（RUN0007） |
| microSD SPI | CS=GPIO21、SCK=D8、MISO=D9、MOSI=D10 | GNSSロガー／UARTロガー |

## 実装順序

### 1. 通信・記録側の統合プロジェクトを作る

既存の `xiao-boat-telemetry-sd-logger` はUART→SD保存成功の基盤として保持する。既存の単体成功コードを壊さないため、通信・記録側の全体役割は新しい統合プロジェクト（仮称 `xiao-boat-telemetry-integration`）で作る。

最初に次を同居させる。

- 実証済みのUART受信専用タスク、フレームキュー、SD書込みバッファ
- GNSS受信・NMEA解析・RAW NMEA記録
- 比較用BNO08X取得
- SoftAP Web UI、JSON状態API、ログ開始／停止
- 制御側からのテレメトリーと通信側独自センサを同じセッションへ保存

この段階では、既存のUART→SD保存経路を変更せず、GNSS/BNOの追加負荷で保存が崩れない構造にする。

### 2. 制御側に最小の受信経路を追加する

`xiao-boat-control-integration` に、通信側からのフレーム受信を追加する。最初に通す操作は `STOP`、`E_STOP`、Heartbeat、状態参照だけとする。設定変更、VESC Duty変更、ウェイポイント変更は停止中でも後段にする。

目的は、通信側Web UIから安全に停止できる全体経路を早期に成立させること。

### 3. 全体のスモーク試験

2台を接続し、次が一度に動くことだけを確認する。

- 制御側のBNO08X、VL53L5CX、INA226、VESC状態が通信側SDへ記録される
- 通信側のGNSS、比較BNO、RAW NMEAが同じ記録セッションへ入る
- 通信側のWeb UIで両ノードの状態・ログ状態が見える
- 通信側からSTOP/E-STOPを送ると、制御側がDuty 0とPCA9685 Full OFFへ遷移する

このスモーク試験では、センサHzや最大間隔の合否判定を完璧にしない。全経路の接続失敗、ピン競合、電源干渉、SD負荷、UART相互通信の大きな問題を早く発見する。

### 4. 統合計測と段階試験

全体形が動いた後に、RUN0007をUART→SD保存の基準として、A〜Fの段階試験、時刻同期、詳細なHz／最大間隔／エラー解析、制御周期決定へ進む。

## 今は行わないこと

- 既存の単体試験プロジェクトの変更
- 状態推定、ESKF、ウェイポイント航法、閉ループロール／舵／推進制御
- センサ周期の先行最適化
- 実機未確認の設定値の確定
